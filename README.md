Online Food Ordering – Group_4


Project Title: Online Food Ordering


A project on online food ordering created by group 4 members. Group 4 backend project at KSTU, Computer Science Department.


#Features 

• Home Page (index.php):
Provides a welcoming introduction to the system along with a simple and intuitive navigation bar for easy access to key features such as registration and login.

• Registration Page (register.php):
Allows new users (students) to create an account by entering their personal details. It also supports profile image upload, enabling users to personalize their accounts.

• Login Page (login.php):
Authenticates registered users by verifying their email and password. Includes proper error handling to notify users of incorrect credentials or missing input fields.

• Dashboard (dashboard.php):
Displays personalized information of the logged-in user, including their name, email, and uploaded profile image. Serves as the main user interface after successful login.

• Logout (logout.php):
Securely terminates the user session and redirects the user back to the home or login page, ensuring proper session management.

• User Interface & Styling:
The system features a consistent purple–blue gradient theme with clean, centered forms and a user-friendly layout, enhancing usability and visual appeal.

• Validation & Security:
Implements basic input validation and sanitization to prevent invalid data entry. Provides clear error messages for incorrect login attempts or incomplete form submissions.



#Challenges

• Session Management and Persistence:
Maintaining user sessions across multiple pages required careful implementation to ensure a seamless user experience. Proper session control mechanisms were necessary to keep users authenticated during navigation and to securely terminate sessions upon logout to prevent unauthorized access.

• File Upload Management:
Handling profile image uploads presented challenges in terms of storage and validation. It was important to correctly store uploaded files in the uploads/ directory while implementing validation checks to ensure only appropriate file types were accepted, thereby enhancing system security and reliability.

• User Interface Consistency:
Achieving a consistent and visually appealing design across all pages required deliberate styling efforts. Ensuring that the purple–blue theme was uniformly applied, while also keeping forms well aligned, centered, and responsive across various devices, was a key design consideration.

• Deployment Constraints:
One major limitation of the system is its dependency on a PHP-enabled server environment. Since platforms like GitHub Pages do not support PHP, the application cannot be deployed there and must instead be run on local servers such as XAMPP or WAMP, or on web hosting services that support PHP.
